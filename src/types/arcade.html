types/arcade.ts の解説

このファイルの役割

このファイルはクラスや関数を持たず、型(type)の定義だけをまとめたファイルです。 Arcade Physics(Phaserの物理エンジン)を使うときによく出てくる「型の名前」を、 プロジェクト内で短く・分かりやすく扱えるようにしています。

前提となる用語

型(type)・型エイリアス
TypeScriptでは、値そのものではなく「値の形」を表す型を定義できます。 export type ArcadeSprite = ... のように、既存の長い型に別名(エイリアス)を付けることを 型エイリアスと呼びます。
Dynamic Body(動的ボディ)
Arcade Physicsの中で、動くことができる物理ボディです。 setVelocity で速度を変えたり、重力の影響を受けたりします。 Player や Bombs、Stars のSpriteはこちらに該当します。 (対になる「Static Body」は Platforms.ts で使われる、動かない地面用のボディです。)
コールバック関数
「何かが起きたときに呼び出される関数」のことです。 physics.add.overlap や physics.add.collider に渡す関数は、 「2つのオブジェクトが重なった/衝突したときに呼ばれるコールバック関数」になります。

このファイルで定義している型

ArcadeSprite
export type ArcadeSprite = Phaser.Types.Physics.Arcade.SpriteWithDynamicBody;
「Dynamic Bodyを持つSprite」を表す型です。 Player.sprite、Bombs や Stars が生成するSpriteなど、 physics.add.sprite や group.create で作られたSpriteの型として使われます。 この型がついていることで、setVelocityX や body.touching.down など 物理用のプロパティ・メソッドにTypeScriptの補完・型チェックが効くようになります。
ArcadeCallback
export type ArcadeCallback = Phaser.Types.Physics.Arcade.ArcadePhysicsCallback;
physics.add.collider や physics.add.overlap の第3引数に渡す コールバック関数の型です。ScrollScene.ts の handleCollectStar や handleHitBomb がこの型で定義されています。

TypeScriptの言語仕様

export type(型エイリアスのエクスポート)
export type ArcadeSprite = Phaser.Types.Physics.Arcade.SpriteWithDynamicBody;
type 名前 = 既存の型 という形で、既存の型に新しい名前を付けています。 export を付けることで、他のファイルから import type { ArcadeSprite } from "..." として利用できます。
import type(型だけのインポート)
import type Phaser from "phaser";
「実行時の値としては使わず、型情報としてだけ使う」ことを示すインポートです。 このファイルは型の定義しか行わないため、Phaser 本体(クラスや関数)は不要で、 型情報だけが必要であることが明示されています。
長い型パスへの別名付け
Phaser.Types.Physics.Arcade.SpriteWithDynamicBody のような長い型を、 使うたびに書くのは大変です。ArcadeSprite という短い名前を1か所で定義しておくことで、 他のファイルでは ArcadeSprite と書くだけで済むようになります。

Phaser 4の仕様

Phaser.Types 名前空間
Phaserは Phaser.Types.〜 という名前空間の下に、 Arcade Physicsや入力(キーボード)などに関する型定義を多数用意しています。 このファイルで使っている SpriteWithDynamicBody や ArcadePhysicsCallback もその一部です。
SpriteWithDynamicBody が必要になる理由
通常の Phaser.GameObjects.Sprite の型には、 Arcade Physics用のプロパティ(body、setVelocityX など)が含まれていません。 physics.add.sprite で作成したSpriteは、実際にはこれらのプロパティを持っていますが、 TypeScriptにそれを伝えるために SpriteWithDynamicBody 型を使う必要があります。

設計のポイント: 型を1か所に集める理由

Player.ts、Bombs.ts、Stars.ts、ScrollScene.ts など、 複数のファイルで「物理ボディ付きSprite」や「衝突コールバック」の型が必要になります。

もしこの型を各ファイルで毎回 Phaser.Types.Physics.Arcade.SpriteWithDynamicBody と 書いてしまうと、長くて読みにくく、修正もしにくくなります。 types/arcade.ts に ArcadeSprite / ArcadeCallback として まとめておくことで、各ファイルは 

import type { ArcadeSprite } from "../types/arcade.js";

のように短く書けます。プロジェクト全体で同じ型を使っていることも一目で分かるようになります。
