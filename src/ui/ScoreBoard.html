ScoreBoard.ts の解説

このファイルの役割

画面左上に表示されるスコアを管理するクラスです。 「現在のスコアの数値」と「画面に表示するテキスト」の両方を内部に持ち、 スコアが変わるたびに表示も一緒に更新します。

前提となる用語

Text オブジェクト
画面上に文字列を表示するためのPhaserのGameObjectです。 Spriteが画像を表示するのに対し、Textは文字を表示します。 一度作成した後は setText(...) で表示内容を書き換えられます。

各メソッドの説明と処理の流れ

① constructor(scene) — 生成時に1回だけ
ScrollScene.create() から new ScoreBoard(this) として呼ばれます。 scene.add.text(...) で「score: 0」という文字を画面に表示し、 そのTextオブジェクトを this.text に保持します。 内部のスコア値 this.score は 0 で初期化されています。
② add(points) — スターを獲得するたびに呼ばれる
ScrollScene の handleCollectStar から、 this._scoreBoard.add(SCORE.star) として呼ばれます。
  1. this.score に points を加算する
  2. this.text.setText(...) で表示文字列を最新のスコアに更新する
「数値の更新」と「表示の更新」を1つのメソッドにまとめることで、 呼び出し側が表示の更新を忘れる心配がなくなります。
③ reset() — スコアを初期状態に戻す
this.score を 0 に戻し、表示も「score: 0」に戻します。
現在の ScrollScene からはまだ呼び出されていません。 リスタート機能などを追加する際に使うことを想定して用意されているメソッドです。

TypeScriptの言語仕様

private フィールドの初期値
private score = 0;
クラスのプロパティは、宣言と同時に初期値を与えることができます。 Player.ts の private readonly scene は 「コンストラクタ引数から受け取る」形でしたが、 score はコンストラクタの引数とは関係なく、常に 0 から始まる値なので、 このようにフィールド宣言の時点で初期化しています。
テンプレートリテラル
this.text.setText(`Score: ${this.score}`);
バッククォート(`)で囲んだ文字列の中に ${式} を書くと、 その式の結果を文字列に埋め込めます。 this.score が 30 なら、"Score: 30" という文字列になります。

Phaser 4の仕様

scene.add.text
this.text = scene.add.text(TEXT.scoreX, TEXT.scoreY, "score: 0", {
  fontSize: TEXT.scoreFontSize,
  color: TEXT.scoreColor,
});
指定した座標に、指定した文字列・スタイルでTextオブジェクトを作成し、画面に追加します。 第4引数のスタイルオブジェクトでは、文字色の指定に color プロパティを使います (constants/game.ts の解説でも触れていますが、Phaser 4では fill ではなく color である点に注意してください)。
setText
this.text.setText(`Score: ${this.score}`);
既に作成済みのTextオブジェクトの表示内容を書き換えます。 新しいTextを作り直す必要はなく、同じオブジェクトの文字列だけが更新されます。
