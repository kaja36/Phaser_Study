main.ts の解説

このファイルの役割

このゲーム全体の起点(エントリーポイント)となるファイルです。 「画面の大きさ」「物理エンジンの設定」「最初に表示するScene」などをまとめた設定オブジェクトを作り、 それを使って Phaser.Game を1つだけ生成します。これによりゲームが起動します。

前提となる用語

GameConfig(ゲーム設定オブジェクト)
Phaserのゲームを起動するときに必要な設定をまとめたオブジェクトです。 画面サイズ、レンダリング方法、物理エンジン、最初に使うSceneなどを指定します。
Phaser.Game インスタンス
Phaserのゲーム本体を表すオブジェクトです。 new Phaser.Game(config) を実行した時点で、設定に基づいてゲームループが開始され、 指定されたScene(ここでは ScrollScene)の preload → create → update... という流れが動き出します。

設定内容の説明と処理の流れ

① type: Phaser.AUTO
画面の描画方法(レンダラー)を指定します。 Phaser.CANVAS(Canvas APIを使用)、Phaser.WEBGL(WebGLを使用)、 Phaser.AUTO(WebGLが使えればWebGL、使えなければCanvasを自動選択)から選べます。 通常は Phaser.AUTO で問題ありません。
② width / height: GAME_SIZE.width / GAME_SIZE.height
ゲーム画面の幅と高さ(ピクセル)です。 constants/game.ts の GAME_SIZE から値を取得しています。 Bombs.ts がボムの出現位置を決める際にも、同じ GAME_SIZE を参照しています。
③ physics: { default: 'arcade', arcade: { gravity, debug } }
使用する物理エンジンを指定します。default: 'arcade' は 「Arcade Physicsを使う」という指定です。 arcade.gravity と arcade.debug には、 constants/game.ts の PHYSICS の値を渡しています。 debug: true にすると、当たり判定の範囲が画面上に線で表示されるようになります (動作確認時に便利です)。
④ input: { keyboard: true }
キーボード入力を有効にします。これにより ScrollScene.create() 内で this.input.keyboard.createCursorKeys() が使えるようになります。
⑤ scene: ScrollScene
最初に起動するSceneとして、ScrollScene クラスそのものを指定します。 インスタンス(new ScrollScene())ではなく、クラス自体を渡している点に注意してください (詳しくはTypeScriptの言語仕様セクションで解説します)。
⑥ new Phaser.Game(config)
ここまでの設定を渡して Phaser.Game のインスタンスを作成します。 この1行が実行された瞬間に、Phaserのエンジンが起動し、 ScrollScene.html で解説しているライフサイクル (preload → create → update)が動き始めます。

TypeScriptの言語仕様

型注釈付きのオブジェクトリテラル
const config: Phaser.Types.Core.GameConfig = {
  type: Phaser.AUTO,
  width: GAME_SIZE.width,
  ...
};
: Phaser.Types.Core.GameConfig という型注釈を付けることで、 config オブジェクトが持つべきプロパティ名・型が決まります。 存在しないプロパティを書いたり、型の合わない値を入れたりすると、その場でエラーとして検出できます。
オブジェクトのネスト(入れ子)
physics: {
  default: 'arcade',
  arcade: {
    gravity: PHYSICS.gravity,
    debug: PHYSICS.debug
  }
}
オブジェクトの値として、さらにオブジェクトを書くことができます。 config.physics.arcade.gravity のように、設定が階層構造で表現されています。
クラスそのものを値として渡す
scene: ScrollScene,
new ScrollScene() ではなく、ScrollScene という クラス自体を値として渡しています。 TypeScript/JavaScriptではクラスも「値」の一種であり、変数に入れたり関数に渡したりできます。 Phaser側がこのクラスを受け取り、必要なタイミングで内部的に new してインスタンス化します。

Phaser 4の仕様

Phaser.AUTO / Phaser.CANVAS / Phaser.WEBGL
Phaserが提供する定数で、レンダリング方式を表します。 type プロパティにこれらの定数を指定することで、描画の仕組みを切り替えられます。
arcade.gravity の形式
constants/game.ts の解説でも触れていますが、 gravity は { x, y } という形のオブジェクトで指定します。 ここで設定した重力は、ゲーム内のすべてのArcade Physicsオブジェクト (プレイヤー、スター、ボムなど)に共通で適用されます。
scene にクラスを渡す仕組み
scene プロパティには、Sceneのクラスを1つ、または配列で複数渡すことができます。 複数渡した場合は、配列の最初のSceneが最初に起動します。 このプロジェクトでは ScrollScene 1つだけを指定しているため、 ゲーム開始時に ScrollScene が起動します。
