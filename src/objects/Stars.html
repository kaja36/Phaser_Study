Stars.ts の解説

このファイルの役割

プレイヤーが集める「スター」を管理するクラスです。 複数のスターを並べて配置し、回収・全回収判定・再配置(復活)までをまとめて担当します。

前提となる用語

Group作成時の key / repeat / setXY
physics.add.group({ key, repeat, setXY }) のように設定オブジェクトを渡すと、 最初から複数のSpriteが並んだ状態のGroupを一度に作成できます。 key は使用する画像、repeat は最初の1個に加えて追加で生成する数、 setXY は配置開始位置と、1個ごとの間隔(stepX)です。 (Bombs.ts では引数なしの physics.add.group() で空のGroupを作っていましたが、 こちらは「最初から並んだ状態」を作る点が異なります。)
getChildren()
Group内に含まれる、すべての子オブジェクト(Sprite)を配列で取得します。 全要素に同じ処理をしたいときに getChildren().forEach(...) という形でよく使われます。
active な状態
PhaserのGameObjectが持つフラグで、「現在ゲーム内で有効かどうか」を表します。 disableBody で無効化されたスターは active = false になり、 countActive(true) でのカウント対象から外れます。

各メソッドの説明と処理の流れ

① constructor(scene) — 生成時に1回だけ
ScrollScene.create() から new Stars(this) として呼ばれます。
  1. STAR_FIELD の設定に従い、複数のスターが並んだ状態のGroupを作成する
  2. getChildren().forEach で各スターを1つずつ取り出し、 setBounceY でランダムな縦方向の跳ね返りを個別に設定する
この時点で、画面上には複数のスターが等間隔に並んだ状態になります。
② collect(star) — プレイヤーがスターに触れたときに1個ずつ
ScrollScene の handleCollectStar から、触れたスター1個を引数として呼ばれます。 disableBody(true, true) により、そのスターの当たり判定を無効化し、画面からも非表示にします。 Group自体から削除するわけではなく、「非表示・無効状態」になる点に注意してください。
③ hasActiveStars() — 全回収判定
collect が呼ばれるたびに ScrollScene から呼ばれ、 まだ表示中(active)のスターが1つでも残っているかを boolean で返します。 false が返ったとき、ScrollScene は「全部回収された」と判断します。
④ reset() — 全回収後の再配置
hasActiveStars() が false を返したときに、ScrollScene から呼ばれます。 すべてのスターに対して enableBody(true, star.x, STAR_FIELD.startY, true, true) を呼び、 「x座標は今の位置のまま、y座標だけ画面上端(STAR_FIELD.startY)に戻して再表示」します。 これにより、スターが画面上から再び降ってくるように見えます。
全体の流れ: ①でスターが並べて配置され、プレイヤーが触れるたびに②collectで1つずつ消えていきます。 ③hasActiveStarsで全て消えたことが分かると、④resetで一斉に再配置され、再び降ってくる動きが繰り返されます。

TypeScriptの言語仕様

as による型アサーション(再掲)
this.group.getChildren().forEach((child) => {
  const star = child as ArcadeSprite;
  ...
});
getChildren() が返す要素の型は汎用的な GameObject です。 Bombs.ts と同様に、as ArcadeSprite で 「物理ボディ付きSpriteとして扱ってよい」ことをTypeScriptに伝えています。
boolean を返すメソッド
hasActiveStars(): boolean {
  return this.group.countActive(true) > 0;
}
countActive(true) は数値(number)を返しますが、 > 0 という比較式の結果は true / false の boolean になります。戻り値の型注釈 : boolean と一致している必要があります。

Phaser 4の仕様

getChildren().forEach(Phaser 4での書き方)
this.group.getChildren().forEach((child) => { ... });
古いバージョンのPhaserでは group.children.iterate(child => { ... }) という書き方が 紹介されることがありますが、Phaser 4では getChildren() で配列を取得し、 通常の配列メソッド(forEach など)を使う書き方が基本になります。
setBounceY / Phaser.Math.FloatBetween
star.setBounceY(Phaser.Math.FloatBetween(STAR_FIELD.minBounceY, STAR_FIELD.maxBounceY));
setBounceY は縦方向だけの跳ね返り具合を設定します。 Phaser.Math.FloatBetween(min, max) は、Phaser.Math.Between(整数)と違い、 指定範囲内のランダムな小数を返します。スターごとに跳ね方が微妙に異なるのはこのためです。
disableBody(true, true) と enableBody(...)
star.disableBody(true, true);
star.enableBody(true, star.x, STAR_FIELD.startY, true, true);
disableBody(disableBody, hideGameObject) の第1引数 true は物理判定を無効化、 第2引数 true は表示も非表示にすることを意味します。 enableBody(reset, x, y, enableGameObject, showGameObject) は逆に、 指定した座標に移動させながら物理判定と表示を有効化(再表示)します。
countActive(true)
countActive(true) は、Group内で active = true になっている (=disableBodyされていない)子オブジェクトの数を返します。
