Bombs.ts の解説

このファイルの役割

プレイヤーに当たるとゲームオーバーになる「ボム」を管理するクラスです。 ボムをまとめるGroupの作成と、新しいボムを出現させる処理を担当します。

前提となる用語

Group(グループ)
複数のSpriteをまとめて管理するためのPhaserの仕組みです。 Group全体に対して衝突判定を設定したり、新しいSpriteを追加したりできます。 Player.ts では1体だけのSpriteを扱いますが、ボムは「複数出現する可能性がある」ため、 最初から空のGroupとして用意しておきます。
Phaser.Math.Between
指定した範囲内のランダムな整数を返すPhaserのユーティリティ関数です。 Phaser.Math.Between(0, 100) なら 0〜100 のいずれかの整数が返ります。

各メソッドの説明と処理の流れ

① constructor(scene) — 生成時に1回だけ
ScrollScene.create() から new Bombs(this) として呼ばれます。 まだボムは1つも生成せず、空のGroup(this.group)だけを用意します。
② spawnFromPlayer(player) — スターを全て回収したときに呼ばれる
ScrollScene の handleCollectStar から呼ばれます。
  1. プレイヤーのx座標が画面の左半分・右半分のどちらにいるかを判定する
  2. プレイヤーがいない側の範囲から、ランダムなx座標を選ぶ
  3. そのx座標と画面上端付近のy座標(BOMB.startY)を使って spawn() を呼ぶ
「プレイヤーの逆側からボムが出てくる」という挙動は、この①②の組み合わせで実現されています。
③ spawn(x, y) — 指定した座標にボムを1つ生成する
  1. group.create(x, y, ASSET_KEYS.bomb) でボムのSpriteをGroupに追加する
  2. setBounce で、壁や地面に当たったときの跳ね返り具合を設定する
  3. setCollideWorldBounds(true) で、画面外に出ないようにする
  4. setVelocity(x方向はランダム, y方向は固定値) で初速を与える
spawnFromPlayer から呼ばれる以外に、このメソッド自体は座標を指定して 単独でも呼び出せる作りになっています。

TypeScriptの言語仕様

三項演算子(条件 ? 真の場合 : 偽の場合)
const x = player.x < GAME_SIZE.width / 2
  ? Phaser.Math.Between(GAME_SIZE.width / 2, GAME_SIZE.width)
  : Phaser.Math.Between(0, GAME_SIZE.width / 2);
条件 ? Aの場合の値 : Bの場合の値 という書き方で、 if文よりも短く「条件によって値を切り分ける」ことができます。 ここでは「プレイヤーが画面左半分にいるか」によって、ボムを出す範囲(右半分 or 左半分)を切り替えています。
as による型アサーション
const bomb = this.group.create(x, y, ASSET_KEYS.bomb) as ArcadeSprite;
group.create(...) の戻り値の型は、Phaserの型定義上は汎用的な Phaser.GameObjects.GameObject になっています。 しかし実際には物理ボディ付きのSpriteが返ってくるため、 as ArcadeSprite で「この値はArcadeSprite型として扱ってよい」とTypeScriptに伝えています。 これにより、続く setBounce や setVelocity などのメソッドを型エラーなく呼び出せます。

Phaser 4の仕様

physics.add.group()
this.group = scene.physics.add.group();
引数なしで呼ぶと、空の動的Group(動くオブジェクト用)を作成します。 Stars.ts のように初期配置を指定する physics.add.group({ key, repeat, setXY }) という書き方もありますが、ボムは後から動的に追加するため、ここでは空のGroupから始めます。
group.create
const bomb = this.group.create(x, y, ASSET_KEYS.bomb) as ArcadeSprite;
指定した座標・画像キーで新しいSpriteを作成し、Groupに追加した上で返します。 Platforms.ts でも同名のメソッドが出てきますが、こちらはStatic Groupではなく 動的(Dynamic)なGroupに対する呼び出しです。
setBounce / setCollideWorldBounds / setVelocity
bomb.setBounce(BOMB.bounce);
bomb.setCollideWorldBounds(true);
bomb.setVelocity(Phaser.Math.Between(BOMB.minVelocityX, BOMB.maxVelocityX), BOMB.velocityY);
Player.ts でも登場した setBounce / setCollideWorldBounds に加えて、 setVelocity(x, y) はx・y方向の初速を同時に設定します。 ボムはx方向にランダムな速度、y方向には常に同じ下向きの速度が与えられ、画面内を不規則にバウンドします。
