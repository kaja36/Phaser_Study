constants/game.ts の解説

このファイルの役割

ゲーム全体で使う「数値」や「文字列」をまとめて定義したファイルです。 クラスや関数は持たず、export const ... による定数の定義だけが並んでいます。 他のファイルはここから必要な値をimportして使います。

前提となる用語

マジックナンバー
コードの中に直接書かれた、意味の分かりにくい数値や文字列のことです。 例えば sprite.setVelocityX(160) という記述だけでは、160 が何を表す数値か分かりません。 PLAYER.speed のような名前付きの定数にすることで、意味が伝わりやすくなり、 変更したい場合もこのファイルの1か所を直すだけで済みます。
定数(const)
一度代入したら再代入できない変数です。設定値のように「途中で変わらないはずの値」には const を使うのが基本です。

各定数グループの説明

GAME_SIZE
ゲーム画面の幅と高さ。main.ts のPhaser設定や、Bombs.ts でボムの出現位置を決める際に使われます。
PHYSICS
Arcade Physicsの初期設定。gravity.y が大きいほど、キャラクターやアイテムが強く下に引かれます。main.ts でのみ使用します。
ASSET_BASE_URL / ASSET_KEYS / ASSET_PATHS
画像・スプライトシートを読み込むための設定です。 ASSET_BASE_URL + ASSET_PATHS の組み合わせが実際の画像URLになり、 ASSET_KEYS はPhaser内でその画像を呼び出すための名前(キー)になります。 ScrollScene.preload() で使われます。
ANIMATION_KEYS
プレイヤーのアニメーション名。Player.ts で anims.create / anims.play のキーとして使われます。
PLAYER
プレイヤーの初期位置・跳ね返り・移動速度・ジャンプ力。Player.ts で使われます。
PLATFORM_LAYOUT
足場(地面)を配置する座標とサイズのリスト。Platforms.ts でこのリストをループして足場を生成します。詳細は Platforms.html を参照してください。
STAR_FIELD
収集アイテム「スター」の並べ方と跳ね方。Stars.ts で使われます。詳細は Stars.html を参照してください。
BOMB
ボムの出現位置・跳ね返り・速度の範囲。Bombs.ts で使われます。
SCORE
スコア計算用の値。現在は「スター1個 = 10点」のみ。ScrollScene.ts の handleCollectStar で使われます。
TEXT
画面に表示するテキスト(スコア・GAME OVER)の位置とスタイル。ScoreBoard.ts と ScrollScene.ts で使われます。

TypeScriptの言語仕様

as const
export const PLAYER = {
  startX: 100,
  startY: 450,
  bounce: 0.2,
  speed: 160,
  jumpVelocity: -300,
} as const;
オブジェクトや配列の末尾に as const を付けると、 TypeScriptはその値を「変更されない値」として扱い、より厳密な型を推論します。 例えば as const がない場合、PLAYER.speed の型は単なる number ですが、 as const を付けると 160 という値そのものの型(リテラル型)になります。 これにより、定数を間違った値で上書きしようとするとエラーになります。
配列 + as const(読み取り専用のタプル)
export const PLATFORM_LAYOUT = [
  { x: 400, y: 568, scale: 2 },
  { x: 600, y: 400 },
  { x: 50, y: 250 },
  { x: 750, y: 220 },
] as const;
配列に as const を付けると、要素数や各要素の型(どの要素に scale があるか等)が そのまま固定された型になります。Platforms.ts 側では "scale" in platform という判定で、scale プロパティの有無を型レベルで安全に確認できます。
オブジェクトの値を他ファイルで利用する(export / import)
export const ASSET_KEYS = { ... } as const;
export をつけた定数は、他のファイルで import { ASSET_KEYS } from "../constants/game.js"; のように読み込んで使えます。 1か所で定義した値を複数ファイルで共有する、最も基本的な方法です。

Phaser 4の仕様

gravity の指定形式
export const PHYSICS = {
  gravity: { x: 0, y: 300 },
  debug: false,
} as const;
Arcade Physicsの重力は { x, y } の形で指定します。 y がプラスの値だと、下方向に重力がかかります(Phaserの画面座標は下方向がプラス)。
spritesheet のフレームサイズ
ASSET_KEYS.dude の画像は、ScrollScene.preload() で { frameWidth: 32, frameHeight: 48 } を指定して読み込まれます。 1枚の画像を32×48ピクセルごとに区切り、複数のフレーム(コマ)として扱うための設定です。 ANIMATION_KEYS のアニメーションは、このフレームを組み合わせて作られます。
TextStyle の color
export const TEXT = {
  ...
  scoreColor: "#FFFFFF",
  ...
  gameOverColor: "#ff0000",
} as const;
Phaser 4のテキストスタイルでは、文字色の指定に color プロパティを使います (古いバージョンや一部のスタイルでは fill という名前が使われていたこともあるため、 Phaser 4のドキュメントを参照する際は color であることに注意してください)。

設計のポイント: 設定値を1ファイルに集約する理由

もしこれらの値を各クラスのコードに直接書いてしまうと、 「プレイヤーの速度を変えたい」と思ったときに、複数のファイルを探し回る必要が出てきます。

このプロジェクトでは、見た目や動きに関わる数値・文字列をconstants/game.tsに集約することで、

  • 調整したい値が1か所にまとまっていて見つけやすい
  • 同じ値を複数ファイルで使う場合も、矛盾が起きない
  • 各クラスのコードは「何のための値か」が名前から分かる状態になる

というメリットが得られます。新しい定数を追加する際も、関連するグループ (PLAYER、BOMB など)にまとめて追加していくと、見通しが良くなります。
