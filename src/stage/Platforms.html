Platforms.ts の解説

このファイルの役割

プレイヤーやアイテムが乗る「地面・足場」を生成して管理するクラスです。 constants/game.ts の PLATFORM_LAYOUT に書かれた座標情報をもとに、 足場用の画像を必要な数だけ並べます。

前提となる用語

Static Group(静的グループ)
Arcade Physicsにおける「動かない物理オブジェクトの集まり」です。 重力の影響を受けず、速度も持ちません。地面や壁など、ゲーム中に位置が変わらないものに使います。 (動くオブジェクト用の型については arcade.html の 「Dynamic Body」も参考にしてください。)
refreshBody
Spriteの見た目を setScale などで拡大・縮小したとき、 当たり判定(物理ボディ)のサイズは自動では変わりません。 refreshBody() を呼ぶことで、見た目のサイズに合わせて当たり判定を更新できます。

処理の流れ

このクラスにはconstructorしかありません。ScrollScene.create() から new Platforms(this) として1回だけ呼ばれます。

  1. scene.physics.add.staticGroup() で、空の静的グループを作成し this.group に保持する
  2. PLATFORM_LAYOUT(座標のリスト)を forEach で1件ずつ処理する
  3. 各座標に対して group.create(x, y, ASSET_KEYS.ground) で足場の画像を1つ生成する
  4. その要素に scale が指定されている場合だけ、setScale で拡大し refreshBody で当たり判定を更新する

PLATFORM_LAYOUT には4つの足場の情報が定義されており、結果として4つの足場が画面上に配置されます。

TypeScriptの言語仕様

forEach によるループ
PLATFORM_LAYOUT.forEach((platform) => { ... });
配列の各要素に対して、順番に同じ処理を実行します。 for 文と違い、ループ用の変数(iなど)を自分で管理する必要がありません。
in 演算子によるプロパティの存在チェック
if ("scale" in platform) {
  body.setScale(platform.scale).refreshBody();
}
"プロパティ名" in オブジェクト で、そのオブジェクトが指定したプロパティを持っているかを確認できます。 PLATFORM_LAYOUT の要素には scale を持つものと持たないものが混在していますが、 この if の中では platform.scale が安全に(型エラーなく)参照できます。 TypeScriptが「このifの中ではscaleプロパティが存在する」と認識してくれるためです。
メソッドチェーン
body.setScale(platform.scale).refreshBody();
setScale(...) の戻り値が再び自分自身(body)であるため、 続けて .refreshBody() を呼び出せます。 「body.setScale(...) を実行した結果に対して .refreshBody() を実行する」という意味です。

Phaser 4の仕様

physics.add.staticGroup
this.group = scene.physics.add.staticGroup();
動かない物理オブジェクトをまとめるためのグループを作成します。 ScrollScene 側では、この group を physics.add.collider(player.sprite, platforms.group) のように衝突判定の対象として渡します。
group.create
const body = this.group.create(platform.x, platform.y, ASSET_KEYS.ground);
指定した座標・画像キーで、グループに新しい子オブジェクト(Sprite)を1つ追加して返します。
setScale と refreshBody の組み合わせ
body.setScale(platform.scale).refreshBody();
PLATFORM_LAYOUT の最初の足場(地面)は scale: 2 が指定されており、 画像を2倍に拡大しています。refreshBody() を呼ばないと、 見た目は大きくなっても当たり判定は元のサイズのままになり、プレイヤーが「すり抜ける」ような状態になってしまいます。
