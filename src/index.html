各ファイルの解説一覧

このプロジェクトは、初学者向けにPhaser 4の使い方を学ぶためのサンプルです。 各.tsファイルには最小限のコメントのみを残し、 詳しい説明は同じディレクトリにある対応する.htmlファイルにまとめています。

読み進める順番について: コードはmain.ts → ScrollScene.ts → 各クラス、という順に実行されますが、 解説は下記の順番で読むと、TypeScriptの基本的な書き方(型・クラスなど)から Phaser 4特有の仕組みへと無理なくつながるようにしています。

1. 土台となる型・定数

types/arcade.html
プロジェクト全体で使う型(ArcadeSprite / ArcadeCallback)の定義。型エイリアスやimport typeの基本がわかります。
constants/game.html
ゲーム全体で使う数値・文字列の定数定義。as constの使い方と、定数を1ファイルに集約する理由がわかります。

2. ゲームを構成する各クラス(委譲される側)

character/Player.html
プレイヤーキャラクターの見た目と操作を管理するクラス。 「継承ではなく委譲」という、このプロジェクト全体の設計の考え方が最初に出てくるページです。
stage/Platforms.html
地面・足場を生成するクラス。Static GroupとrefreshBodyについて説明しています。
objects/Stars.html
収集アイテム「スター」を管理するクラス。Groupの一括生成・回収・再配置の流れがわかります。
objects/Bombs.html
敵オブジェクト「ボム」を管理するクラス。動的なGroupへのSprite追加と型アサーション(as)について説明しています。
ui/ScoreBoard.html
スコア表示を管理するクラス。Textオブジェクトの基本的な扱い方がわかります。

3. 全体の調整役(継承する側)

scenes/ScrollScene.html
Phaser.Sceneを継承するSceneクラス。preload/create/updateのライフサイクル、 各クラスへの「委譲」、衝突・接触判定の設定方法を解説しています。 このプロジェクトの設計を理解する上で最も重要なページです。

4. ゲームの起動

main.html
ゲーム全体のエントリーポイント。Phaser.Gameの設定項目と起動の流れを説明しています。

このプロジェクトの設計のまとめ

このプロジェクトは、次の2つの考え方を組み合わせて作られています。

  • 継承: ScrollSceneだけがPhaser.Sceneを継承し、Phaserのライフサイクル(preload/create/update)に乗る
  • 委譲: ゲームの各要素(プレイヤー、足場、アイテム、スコアなど)は、Phaserのクラスを継承しないただのTypeScriptクラスとして作り、ScrollSceneからインスタンスを生成して処理を任せる

別のプロジェクトで新しい機能(新しいキャラクターやアイテムなど)を追加する場合も、 この「委譲されるクラスを1つ作り、ScrollScene(に相当するSceneクラス)から呼び出す」 という形を踏襲すると、Sceneクラスが肥大化しにくく、見通しの良い構成を保てます。
